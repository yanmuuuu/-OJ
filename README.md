# OnlineJudge - 在线OJ判题系统

## 项目简介

OnlineJudge（在线判题系统）是一个模仿 LeetCode 的在线编程平台，支持题目浏览、代码在线编写、代码编译运行以及结果展示等功能。

项目采用前后端分离思想，实现了代码编译服务与业务服务解耦，并通过负载均衡策略实现多个编译服务器协同工作，提高系统并发处理能力。

## 项目功能

### 题库管理

* 查看题目列表
* 查看题目详情
* 展示题目描述

### 在线编程

* 在线编写代码
* 在线提交代码
* 在线编译运行代码
* 返回编译结果
* 返回运行结果

### 判题服务

* 编译用户代码
* 执行用户程序
* 获取标准输出
* 获取错误信息
* 返回统一格式结果

### 负载均衡

* 多台编译服务器同时工作
* 根据负载情况分配任务
* 支持服务器动态上线与下线
* 提高系统吞吐能力

## Compile Server设计

Compile Server负责代码编译与运行。

### 输入

用户提交的：

* 源代码
* 输入数据

### 处理流程

```text
接收请求
    │
    ▼
生成临时文件
    │
    ▼
调用g++编译
    │
    ▼
编译成功?
 ┌──┴──┐
 │     │
否     是
 │     │
 ▼     ▼
返回错误 运行程序
           │
           ▼
       获取结果
           │
           ▼
       返回JSON
```

### 输出格式

```json
{
    "status": 0,
    "reason": "success",
    "stdout": "hello world",
    "stderr": ""
}
```

## 核心技术

将业务逻辑与代码编译运行逻辑拆分为独立服务：

* 高内聚
* 低耦合
* 易于扩展

### 负载均衡设计

维护编译服务器负载信息：

* 当前任务数
* 权重信息
* 在线状态

动态选择负载最低服务器处理请求。

### 多线程并发处理

采用线程池思想处理大量HTTP请求，提高系统并发能力。

### 数据持久化

利用MySQL管理题目信息，实现：

* 数据统一管理
* 快速查询
* 方便扩展
